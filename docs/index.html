Overview

AgentGuard is a Human-as-a-Service (HaaS) governance layer that sits between AI agents and their tool calls. It intercepts every MCP tool invocation, classifies risk, and routes high-risk actions to qualified human reviewers for approval before execution.

🛡️

Risk Classification

Policy-driven 4-tier system: LOW → MID → HIGH → CRITICAL. Each tier has configurable approval thresholds.

👥

Multi-Sig Approval

Cross-pool consensus with threshold-based quorum. Higher risks need more approvers from diverse pools.

🔐

Credential Verification

Approvers authenticate via LinkedIn, GitHub, SSO, or OIDC. Skills are verified before review access.

🧠

AIOps Learning

Learns from approval patterns to auto-adjust risk tiers. Consistently-approved tools get downgraded.

📡

MCP Protocol

Official MCP SDK server exposing governance tools. Drop-in integration with Claude, Cursor, and others.

📊

Admin Dashboard

Light-themed admin console for queue monitoring, policy management, and audit log review.

AgentGuard operates as a transparent proxy between AI agents and tool servers. The core intercepts tool calls, classifies risk, and manages the approval lifecycle.

Admin Dashboard (packages/haas-dashboard)

React + Vite + Tailwind admin console. Monitors governance queue, manages policies and identity providers, and provides shareable approval links for SMEs.

Standalone page for SME/specialist review. Three-step flow: Authenticate → Verify Skills → Review & Decide. Accessed via shareable links generated from the dashboard.

Request Flow

Every tool call from an AI agent flows through the following lifecycle:

sequenceDiagram participant Agent as AI Agent participant MCP as MCP Interceptor participant RC as Risk Classifier participant SM as State Machine participant Human as Human Reviewer participant Tool as Tool Server Agent->>MCP: tools/call (transfer_funds) MCP->>RC: classify(toolName, args) RC-->>MCP: CRITICAL (threshold: 3) alt LOW Risk MCP->>Tool: Execute immediately Tool-->>Agent: Result else MID / HIGH / CRITICAL MCP->>SM: createRequest(request, policy) SM-->>MCP: WAITING_FOR_HUMAN Note over SM: Request queued loop Until threshold met or timeout Human->>SM: addSignature(requestId, sig) SM-->>SM: Check consensus end alt Consensus Reached SM-->>MCP: APPROVED MCP->>Tool: Execute Tool-->>Agent: Result else Rejected SM-->>MCP: REJECTED MCP-->>Agent: Error (rejected) else Timeout SM-->>MCP: TIMEOUT MCP-->>Agent: Error (timeout) end end

Approval Flow

When a request needs human approval, the admin dashboard generates a shareable link. The SME/specialist follows this flow:

sequenceDiagram participant Admin as Admin Dashboard participant SME as SME / Specialist participant AP as Approval Page participant IdP as Identity Provider participant SF as Skill Framework participant SM as State Machine Admin->>SME: Share approval link SME->>AP: Open /approve/:requestId AP->>AP: Load request details AP->>IdP: Authenticate (LinkedIn/GitHub/SSO) IdP-->>AP: Claims + credentials AP->>SF: Verify skills match pool alt Skills Match SF-->>AP: ✅ Qualified AP->>SME: Show Review & Decide SME->>AP: Approve / Reject AP->>SM: submitSignature() SM-->>AP: Decision recorded else Skills Mismatch SF-->>AP: ❌ Not qualified AP->>SME: Show blocked message end

Risk Classification

Every tool call is classified into one of four tiers based on configurable policies:

TierThresholdDefault TimeoutPoolsExample
LOWAuto-approve——read_customer_data
MID1 signature3 mingeneralupdate_customer_notes
HIGH2 signatures5 minfinanceapply_discount
CRITICAL3 signatures15 minfinance + security + legaltransfer_funds

Policy Configuration

Policies are defined as JSON arrays, editable via the dashboard Settings page or loaded from config/risk-policies.example.yml:

[
  {
    "tool": "transfer_*",
    "tier": "CRITICAL",
    "threshold": 3,
    "pools": ["finance", "security", "legal"],
    "timeout": "15m",
    "conditions": [
      { "field": "amount", "operator": ">", "value": 10000 }
    ]
  },
  {
    "tool": "read_*",
    "tier": "LOW",
    "autoApprove": true
  }
]

AIOps Feedback Loop

AgentGuard learns from approval patterns and progressively adjusts risk tiers to reduce friction for consistently-approved operations.

graph LR A[Tool Call] --> B[Risk Classifier] B --> C{Approval Decision} C -->|Approved| D[Record Approval] C -->|Rejected| E[Record Rejection] D --> F{50+ consecutive approvals?} F -->|Yes| G[Recommend Downgrade] G --> H{Is CRITICAL tier?} H -->|Yes| I[Block - Protected Tier] H -->|No| J[Auto-downgrade tier] E --> K{5+ consecutive rejections?} K -->|Yes| L[Auto-upgrade tier] F -->|No| M[Continue tracking] K -->|No| M

Safety Guardrails

  • CRITICAL tier is protected — never auto-downgraded, only flagged for manual review
  • Configurable thresholds — default: 50 approvals to downgrade, 5 rejections to upgrade
  • Full audit trail — every adjustment is logged with timestamp and reason
  • Manual override — admins can lock any tool's tier via the dashboard

Quickstart

Clone & Install

git clone https://github.com/anthropics/agentguard.git
cd agentguard
npm install

Build

cd packages/shared-proto && npx tsc && cd ../..
cd packages/haas-core && npx tsc && cd ../..

Start Everything

Use the deploy script to start the API server + dashboard:

./deploy-mcp.sh

Open Dashboard

Visit http://localhost:5173 for the admin dashboard.

Trigger Demo

Click "Trigger Demo" in the dashboard, or run:

./deploy-mcp.sh demo

Test Approval Flow

Click "Open" next to a pending request → authenticate → verify skills → approve or reject.

Connect MCP Client

For Claude Desktop or Cursor, add to your mcp_config.json:

{
  "mcpServers": {
    "agentguard": {
      "command": "npx",
      "args": ["tsx", "packages/haas-core/src/mcp/mcp-server.ts"],
      "cwd": "/path/to/agentguard"
    }
  }
}

Persistent Data

Settings and audit logs are stored in .agentguard/ directory:

  • .agentguard/providers.json — Identity provider configuration
  • .agentguard/audit-log.json — Governance decision history
  • .agentguard/policies.json — Risk policy overrides

REST API Reference

All endpoints are served from the HaaS Core HTTP server (default port 3100).

Core Endpoints

POST /mcp/tools/call

Intercept a tool call. Body: { method, params: { name, arguments } }

POST /mcp/approve

Submit approval/rejection. Body: { requestId, signature: { approverId, pool, decision, reason } }

GET /pending

List all pending approval tasks.

GET /health

Health check. Returns { status: "ok", uptime }

Settings Endpoints

GET /settings/policies

Get current risk policies.

POST /settings/policies

Update risk policies. Body: { policies: [...] }

GET /settings/providers

Get all identity provider settings.

POST /settings/providers

Save identity provider settings. Body: { providers: [{ name, enabled }] }

GET /settings/providers/enabled

Get only enabled providers (used by approval page).

Monitoring Endpoints

GET /aiops/stats

AIOps learning statistics: tracked tools, adjustments, pending recommendations.

GET /audit/log

Governance audit log (last 100 entries from persistent storage).

POST /demo/trigger

Trigger demo scenarios. Body: { scenario: "all" | "low" | "mid" | "high" | "critical" }
